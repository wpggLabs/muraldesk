Goal:

PART 2 — Fix the “stuck in a box” movement issue
- Items can only move inside the Electron BrowserWindow. Make sure Desktop Mode expands the BrowserWindow to the full current display.
- When Electron Desktop Mode is enabled:
  - use the display bounds for the monitor the window is on
  - set the BrowserWindow bounds to cover that full display
  - renderer root/canvas should be fixed/absolute inset: 0
  - html/body/#root should be 100vw/100vh
  - no max-width, padding, centered container, or hidden parent reducing item movement
  - react-rnd parent bounds should be the full display-sized canvas
- When Desktop Mode is OFF:
  - normal window mode can stay bounded to the normal window.
- Make the Desktop button label clearer:
  - “Desktop” when not active
  - “Exit Desktop” when active
- If mural.desktopMode is true on launch, open directly as full-display overlay, not a small remembered box.
- Do not save full-display overlay bounds as normal window geometry.
